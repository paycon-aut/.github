# Politica de Seguranca - Paycon Automacoes

## Versoes suportadas

| Versao | Suporte          |
|--------|------------------|
| main   | Suporte ativo    |
| develop| Suporte ativo    |

## Reportando vulnerabilidades

Se voce encontrou uma vulnerabilidade de seguranca, **NAO abra uma issue publica**.

Envie um e-mail para: **breno.dias@paycon.com.br**

Inclua:

- Descricao da vulnerabilidade
- Passos para reproduzir
- Impacto estimado
- Sugestao de correcao (se tiver)

### Prazo de resposta

- **Confirmacao de recebimento:** ate 48 horas uteis
- **Avaliacao inicial:** ate 5 dias uteis
- **Correcao (critico):** ate 7 dias uteis
- **Correcao (alto):** ate 14 dias uteis

## Praticas de seguranca

A Paycon Automacoes segue as seguintes praticas:

- MFA obrigatorio para todos os membros da organizacao
- Branch protection em main e develop
- Code review obrigatorio para todo merge
- Secret scanning ativo
- Dependabot habilitado
- Auditoria de acessos periodica
- Principio de menor privilegio

## Conformidade

Esta estrutura esta alinhada com os requisitos de:

- ISO 27001
- SOC 2 Type II

Para duvidas sobre seguranca, entre em contato com o time de Infra & Security.
